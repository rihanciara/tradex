## 📜 Version History​

A comprehensive guide for automatically adding all child products when a combo product is selected in the POS screen.

## 📋 Overview​

This feature automatically adds all individual products within a combo product as separate line items when the combo is selected in POS, eliminating the need for manual product addition. The feature includes a **business setting** to enable or disable this behavior, allowing businesses to choose their preferred workflow.

### Features​

  * ✅ Auto-detect combo products in search
  * ✅ Add all child products as individual rows
  * ✅ Proper quantity calculation per child
  * ✅ Works with product search autocomplete
  * ✅ Works with product box clicks
  * ✅ Maintains stock tracking for each component
  * ✅ **Business setting to enable/disable feature**
  * ✅ **Enabled by default for new installations**

```php
public function getProductRow($variation_id, $location_id)
{
    $output = [];

    try {
        $row_count = request()->get('product_row');
        $row_count = $row_count + 1;
        $quantity = request()->get('quantity', 1);
        $weighing_barcode = request()->get('weighing_scale_barcode', null);

        $is_direct_sell = false;
        if (request()->get('is_direct_sell') == 'true') {
            $is_direct_sell = true;
        }

        $is_serial_no = false;
        if (request()->get('is_serial_no') == 'true') {
            $is_serial_no = true;
        }

        if ($variation_id == 'null' && !empty($weighing_barcode)) {
            $product_details = $this->__parseWeighingBarcode($weighing_barcode);
            if ($product_details['success']) {
                $variation_id = $product_details['variation_id'];
                $quantity = $product_details['qty'];
            } else {
                $output['success'] = false;
                $output['msg'] = $product_details['msg'];

                return $output;
            }
        }

        // Check if combo product - add child products instead (only if enabled in settings)
        $business_id = request()->session()->get('user.business_id');
        $variation = Variation::find($variation_id);
        $product = Product::where('business_id', $business_id)->find($variation->product_id);
        $business_details = $this->businessUtil->getDetails($business_id);
        $pos_settings = empty($business_details->pos_settings) ? $this->businessUtil->defaultPosSettings() : json_decode($business_details->pos_settings, true);

        if ($product->type == 'combo' && !empty($variation->combo_variations) && !empty($pos_settings['enable_combo_product_auto_addition'])) {
            // Add all child products
            $output['success'] = true;
            $output['html_content'] = '';
            $output['is_combo'] = true;
            $output['combo_count'] = 0;

            foreach ($variation->combo_variations as $combo_item) {
                $child_quantity = $combo_item['quantity'] * $quantity;
                $child_output = $this->getSellLineRow($combo_item['variation_id'], $location_id, $child_quantity, $row_count, $is_direct_sell, $is_serial_no);

                if ($child_output['success']) {
                    $output['html_content'] .= $child_output['html_content'];
                    $row_count++;
                    $output['combo_count']++;
                }
            }
        } else {
            $output = $this->getSellLineRow($variation_id, $location_id, $quantity, $row_count, $is_direct_sell, $is_serial_no);

            if ($this->transactionUtil->isModuleEnabled('modifiers') && !$is_direct_sell) {

                    $output['html_modifier'] = view('restaurant.product_modifier_set.modifier_for_product')
                        ->with(compact('product_ms', 'row_count'))->render();
                }
            }
        }
    } catch (\Exception $e) {
        \Log::emergency('File:' . $e->getFile() . 'Line:' . $e->getLine() . 'Message:' . $e->getMessage());

        $output['success'] = false;
        $output['msg'] = __('lang_v1.item_out_of_stock');
    }

    return $output;
}
```

## 💡 How It Works​

### Business Setting Control​

The feature is controlled by a business setting called 
```text
enable_combo_product_auto_addition
```
:


```text
Location: Business Settings → POS tab
Default: Enabled (1) for new installations
Storage: business.pos_settings JSON column
```


**When Enabled:**

  * Combo selection auto-adds all children as separate rows
  * Traditional single-row combo behavior is bypassed



**When Disabled:**

  * Combo products behave as regular products (single row)
  * Users can manually add children if needed



### Dual-Path Architecture​

**Path 1: Autocomplete (Frontend)**


```text
User searches → Check pos_settings.enable_combo_product_auto_addition
→ If enabled: Autocomplete detects type='combo'
→ Parse combo_variations JSON → Loop children
→ Call pos_product_row() for each → Individual rows added
```


**Path 2: Product Box Click (Backend)**


```text
User clicks box → Backend loads pos_settings
→ Check enable_combo_product_auto_addition setting
→ If enabled: getProductRow() detects combo
→ Loop children → Generate HTML for all → Return concatenated
→ Frontend processes multiple rows at once
```


### Data Flow​


```text
// Combo variations structure in database
{
  "combo_variations": [
    {
      "variation_id": 123,
      "product_id": 45,
      "quantity": 2,
      "unit_id": 1
    },
    {
      "variation_id": 456,
      "product_id": 78,
      "quantity": 1,
      "unit_id": 1
    }
  ]
}
```


Each child is added with 
```text
quantity = combo_item.quantity * parent_quantity
```
.

## 🔍 Troubleshooting​

### Issue: Combo products not being auto-added​

**Solution:**

  * Check Business Settings → POS tab → "Enable Combo Product Auto-Addition"
  * Ensure the setting is **checked/enabled**
  * Clear browser cache
  * Run 
```text
php artisan cache:clear
```

  * Verify combo_variations is not null in database
  * Check product type is exactly 'combo' in products table



### Issue: Setting not saving properly​

**Solution:**

  * Verify language strings are added to 
```text
lang/en/lang_v1.php
```

  * Check that 
```text
enable_combo_product_auto_addition
```
 is in 
```text
defaultPosSettings()
```

  * Ensure the checkbox name is: 
```text
pos_settings[enable_combo_product_auto_addition]
```

  * Check for PHP errors in Laravel logs



### Issue: JavaScript error "pos_settings is not defined"​

**Solution:**

  * Verify 
```text
var pos_settings = @json($pos_settings);
```
 is added to 
```text
resources/views/sale_pos/create.blade.php
```

  * Check that 
```text
$pos_settings
```
 is being passed to the view from SellPosController
  * Ensure the view is loading the JavaScript section



### Issue: Child products added with wrong quantity​

**Solution:**

  * Verify combo_variations JSON has correct quantity values
  * Check quantity multiplier logic: 
```text
child.quantity * parent_quantity
```




### Issue: Only one child product added​

**Solution:**

  * Check loop logic in both frontend and backend
  * Verify combo_variations is an array with multiple items
  * Check for JavaScript errors in browser console
  * Verify the setting check: 
```text
pos_settings.enable_combo_product_auto_addition
```
 is true



### Issue: Row numbering incorrect​

**Solution:**

  * Verify 
```text
combo_count
```
 is properly incremented in backend
  * Check 
```text
rows_added
```
 calculation in frontend
  * Ensure 
```text
product_row_count
```
 increments by number of children, not 1



### Issue: Backend not checking setting​

**Solution:**

  * Verify 
```text
$pos_settings
```
 is loaded in 
```text
getProductRow()
```
 method
  * Check: 
```text
$business_details = $this->businessUtil->getDetails($business_id);
```

  * Verify setting check: 
```text
!empty($pos_settings['enable_combo_product_auto_addition'])
```

  1. **Test thoroughly** \- Verify with combos containing 2-5 child products

  3. **Stock validation** \- Test with low stock scenarios
  4. **Performance** \- Avoid combos with 10+ children (causes slow UI)
