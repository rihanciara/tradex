# Combo Product Auto-Addition in POS

A comprehensive guide for automatically adding all child products when a combo product is selected in the POS screen.

## 📋 Overview​

This feature automatically adds all individual products within a combo product as separate line items when the combo is selected in POS, eliminating the need for manual product addition.

### Features​

  * ✅ Auto-detect combo products in search
  * ✅ Add all child products as individual rows
  * ✅ Proper quantity calculation per child
  * ✅ Works with product search autocomplete
  * ✅ Works with product box clicks
  * ✅ Maintains stock tracking for each component
  * ✅ No UI changes needed



* * *

## 🚀 Quick Start​

### Prerequisites​

  * Ultimate POS with combo product module enabled
  * Existing combo products configured
  * Laravel-based Ultimate POS installation



### Compatibility​

  * Works with all Ultimate POS versions supporting combo products
  * Compatible with existing POS workflows
  * No database migrations required

if (ui.item.enable_stock != 1 || ui.item.qty_available > 0 || is_overselling_allowed || for_so || is_draft) {
    $(this).val(null);

    // Check if combo product - add all child products instead
    if (ui.item.type === 'combo' && ui.item.combo_variations) {
        var combo_variations = ui.item.combo_variations;
        if (typeof combo_variations === 'string') {
            try {
                combo_variations = JSON.parse(combo_variations);
            } catch (e) {
                console.error('Failed to parse combo_variations:', e);
                combo_variations = null;
            }
        }

        if (combo_variations && Array.isArray(combo_variations)) {
            for (var i = 0; i < combo_variations.length; i++) {
                var child = combo_variations[i];
                if (child.variation_id && child.quantity) {
                    pos_product_row(child.variation_id, null, null, child.quantity);
                }
            }
        }
    } else {
        //Pre select lot number only if the searched term is same as the lot number
        var purchase_line_id = ui.item.purchase_line_id && searched_term == ui.item.lot_number ? ui.item.purchase_line_id : null;
        pos_product_row(ui.item.variation_id, purchase_line_id);
    }

    alert(LANG.out_of_stock);

            }
        }

        // Check if combo product - add child products instead
        $business_id = request()->session()->get('user.business_id');

        $product = Product::where('business_id', $business_id)->find($variation->product_id);

            // Add all child products

## 💡 How It Works​

### Dual-Path Architecture​

**Path 1: Autocomplete (Frontend)**


```text
User searches → Autocomplete detects type='combo'
→ Parse combo_variations JSON → Loop children
→ Call pos_product_row() for each → Individual rows added
```


**Path 2: Product Box Click (Backend)**


```text
User clicks box → Backend getProductRow() detects combo
→ Loop children → Generate HTML for all → Return concatenated
→ Frontend processes multiple rows at once
```


### Data Flow​


```text
// Combo variations structure in database
{
  "combo_variations": [
    {
      "variation_id": 123,
      "product_id": 45,
      "quantity": 2,
      "unit_id": 1
    },
    {
      "variation_id": 456,
      "product_id": 78,
      "quantity": 1,
      "unit_id": 1
    }
  ]
}
```


Each child is added with 
```text
quantity = combo_item.quantity * parent_quantity
```
.

## 🔍 Troubleshooting​

### Issue: Combo products not being detected​

**Solution:**

  * Clear browser cache
  * Run 
```text
php artisan cache:clear
```

  * Verify combo_variations is not null in database
  * Check product type is exactly 'combo' in products table



### Issue: Child products added with wrong quantity​

**Solution:**

  * Verify combo_variations JSON has correct quantity values
  * Check quantity multiplier logic: 
```text
child.quantity * parent_quantity
```




### Issue: Only one child product added​

**Solution:**

  * Check loop logic in both frontend and backend
  * Verify combo_variations is an array with multiple items
  * Check for JavaScript errors in browser console



### Issue: Row numbering incorrect​

**Solution:**

  * Verify 
```text
combo_count
```
 is properly incremented in backend
  * Check 
```text
rows_added
```
 calculation in frontend
  * Ensure 
```text
product_row_count
```
 increments by number of children, not 1

## 🎯 Best Practices​

  1. **Test thoroughly** \- Verify with combos containing 2-5 child products
  2. **Check permissions** \- Ensure users have access to child products
  3. **Stock validation** \- Test with low stock scenarios
  4. **Performance** \- Avoid combos with 10+ children (causes slow UI)
  5. **Clear documentation** \- Train staff that combos add multiple rows
