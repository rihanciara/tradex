# Product History Button in POS Screen

Add a **Product History** button to the POS header that opens a modal to search products by SKU or name and view their complete history including sales, purchases, stock movements, warranty, and expiry information.

![Product History in POS](/img/docs/product-history-in-pos-1.png)

## Download​

  * [Download ZIP](https://ultimate-pos-tutorials.vercel.app/assets/files/product-history-pos-4a139943ddb99efc5f0cc1a83d3b4475.zip)



## Features​

  * Search by **SKU** or **Product Name** (partial match supported)
  * Search by **Variation SKU** (sub_sku)
  * **5 History Tabs:**
    * **Sales History** \- Invoice, date, customer, qty, price, subtotal, location
    * **Purchase History** \- Ref no, date, supplier, qty, price, subtotal, location
    * **Stock History** \- Date, transaction type, ref, location, qty in/out, balance
    * **Warranty Info** \- Invoice, date, customer, warranty details, status (Active/Expired)
    * **Expiry Info** \- Lot number, expiry date, location, stock, days to expiry, status
  * **Pagination** \- 8 records per page with navigation controls
  * **Bilingual Support** \- English and Arabic translations
  * **Permission-Based Access** \- Each tab respects user permissions



## Permissions​

<table><thead><tr><th>Section<th>Required Permission<tbody><tr><td><strong>Base Access</strong><td><code>product.view</code><tr><td><strong>Sales History</strong><td><code>sell.view</code> or <code>direct_sell.view</code><tr><td><strong>Purchase History</strong><td><code>purchase.view</code><tr><td><strong>Stock History</strong><td><code>product.view</code><tr><td><strong>Warranty Info</strong><td><code>sell.view</code> or <code>direct_sell.view</code><tr><td><strong>Expiry Info</strong><td><code>purchase.view</code></td></td></tr></td></td></tr></td></td></tr></td></td></tr></td></td></tr></td></td></tr></tbody></th></th></tr></thead></table>

 

Users will only see data for tabs they have permission to access. If a user doesn't have the required permission, the tab will show empty data.

## Installation Steps​

### Step 1: Add Button to POS Header​

Edit 
```text

```


Find the calculator button and add the product history button after it:



## Usage​

  1. Go to POS screen
  2. Click the purple **history icon** button in the header
  3. Enter product SKU or name in the search box
  4. Press **Enter** or click **Search**
  5. View history across all tabs (Sales, Purchases, Stock, Warranty, Expiry)
  6. Use pagination controls to navigate through records

## Customization​

### Change Records Per Page​

Edit 
```text
SellPosController.php
```
 line with 
```text
$per_page = 8;
```
 to change the number of records per page.

### Change Button Color​

Edit the button class in 
```text
header-pos.blade.php
```
:

  * Current: 
```text
tw-text-[#9333EA]
```
 (purple)
  * Change to any Tailwind color class



## Troubleshooting​

### Modal Not Opening​

  * Check if the modal include is added to both 
```text
create.blade.php
```
 and 
```text
edit.blade.php
```

  * Clear browser cache and refresh



### No Data Showing​

  * Verify the route is added correctly
  * Check browser console for JavaScript errors
  * Verify controller method is added



### Translations Not Working​

  * Clear Laravel cache: 
```text
php artisan cache:clear
```

  * Check translation keys match exactly
